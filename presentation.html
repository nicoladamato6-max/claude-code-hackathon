What We Fixed — 23 of 36 Findings in Code

Security

  • Critical DEBUG=False — env var, default false
  • Critical pg_hba removed — SG rules only
  • Critical dblink credential — replaced with ETL join
  • Critical reporting_user — 5 per-team roles
  • High SQLi/XSS — WAF + parameterised queries
  • High Cert expiry — ACM auto-renew
  • High pgaudit — enabled in RDS param group

Reliability

  • High DR=zero → RDS Multi-AZ RPO<5min
  • High Redis crash → AOF + Multi-AZ failover
  • High No autoscale → ECS CPU tracking
  • High Patch delays → new image = patch
  • High No healthz → /healthz + ECS replace
  • High Batch silent retry → fail-fast + alarm
  • High Log 7 days → CloudWatch 90d + Glacier

Correctness

  • Critical Batch exit 0 → exit 1 on failures
  • High SECURITY DEFINER → SECURITY INVOKER
  • High Hardcoded IPs → env vars
  • High Filesystem mounts → S3 client
  • High Matview cron → pg_cron on RDS
  • High No structured log → JSON + CloudWatch
  • Medium requirements unpinned → pinned versions

Why replatform enables Phase 2

No renegotiation — AWS contract already permits Lambda, EventBridge, DynamoDB. Phase 2 is zero procurement friction.

Same Docker images — ECS tasks become Lambda containers with no rebuild. The containerisation investment is preserved.

Operational confidence — SRE team gained cloud-ops experience during Phase 1. Serverless debugging is now tractable.

Test suite carries forward — Smoke, contract, data-integrity tests are environment-agnostic. They will validate Phase 2 without rewriting.
